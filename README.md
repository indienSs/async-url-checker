REST API для асинхронной проверки списка URL и фронтенд-приложение для работы с этим API. TypeScript обязателен для фронтенда и бэкенда.

Функционал бэкенда:
1. Создание задания — POST /api/jobs
Принимает JSON:
{"urls": ["https://example.com", "..."]}

При успешном создании:
задание сохраняется с уникальным jobId и статусом pending
асинхронная обработка URL запускается в фоне
в ответе возвращается: { "jobId": "..." }

2. Список заданий — GET /api/jobs
Возвращает список заданий с краткой информацией:
id, createdAt, статус (pending, in_progress, completed, cancelled, failed)
количество URL и статистика (успешно / с ошибкой)

3. Детальная информация — GET /api/jobs/:id
Для каждого URL в задании возвращает:
URL и статус (pending, in_progress, success, error, cancelled)
HTTP-статус (если есть)
сообщение об ошибке (если есть)
время начала/окончания обработки и длительность

4. Отмена задания — DELETE /api/jobs/:id
Помечает задание как cancelled и прекращает обработку не начатых URL.

5. Логика проверки URL
Для каждого URL выполняется HTTP HEAD-запрос
Перед сохранением результата — искусственная задержка (случайно, 0–10 секунд)
Не более 5 одновременных HEAD-запросов на одно задание
Допускается одновременная обработка нескольких заданий

Технические требования:
NestJS, достаточно in-memory хранилища, БД не нужна

Функционал фронтенда:

1. Форма создания задания
textarea, где каждый URL — с новой строки
кнопка "Запустить проверку"
при отправке: POST /api/jobs → полученный jobId становится активным заданием

2. Список заданий
GET /api/jobs — список последних заданий
краткая информация: id, дата, статус, статистика
возможность выбрать задание как активное

3. Детальная информация по активному заданию
общий статус и прогресс (например, «X из Y обработано»)
список URL с их статусами, HTTP-кодами и/или ошибками

4. Опрос статуса и отмена
периодический опрос GET /api/jobs/:id пока статус не конечный
при смене активного задания или создании нового — опрос предыдущего корректно останавливается
ответы по старому jobId не меняют состояние интерфейса
кнопка "Отменить задание" вызывает DELETE /api/jobs/:id

Технические требования:
TypeScript, React, MobX
Приложение должно быть разделено на:
- компоненты
- слой работы с API
- глобальное состояние (список заданий, активное задание, детали, состояния загрузки/ошибок)
